密码管理器vaultwarden

2025年2月11日 · Yangly

摘要：自建vaultwarden，实现个人密码管理。

Vaultwarden，Bitwarden 客户端 API 的替代服务器实现，用 Rust 编写并与官方 Bitwarden 客户端兼容 [免责声明]，非常适合运行官方资源密集型服务可能不理想的自托管部署。

部署请参考：aultwarden Wiki和中文版，建议Docker Compose + Caddy部署，将Vaultwarden 服务器和代理都将请求重定向到正确的地方。

1、安装docker compose，需要提前安装docker。

sudo curl -L "https://github.com/docker/compose/releases/download/1.29.2/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

2、配置docker compose

mkdir ~/vaultwarden
cd ~/vaultwarden

vim compose.yml

需要设置的几个点：（1）vaultwarden下image镜像为1.30.5，最新版本网页端无法登录；（2）vaultwarden下environment的DOMAIN 域名；（3）caddy下environment的DOMAIN域名（和vaultwarden一致），caddy下environment的EMAIL邮件。

services:
  vaultwarden:
    image: vaultwarden/server:1.30.5
    container_name: vaultwarden
    restart: always
    environment:
      DOMAIN: "TODO" # 使用反向代理时必填；您的域名；Vaultwarden 需要知道它是 https 才能正确处理附件
      SIGNUPS_ALLOWED: "true" # 创建账户后，使用 "false" 停用此选项，这样就不会有陌生人注册了
    volumes:
      - ./vw-data:/data # : 前面的路径可以修改
  caddy:
    image: caddy:2
    container_name: caddy
    restart: always
    ports:
      - 80:80  #  ACME HTTP-01 验证需要
      - 443:443
      - 443:443/udp # HTTP/3 需要
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile:ro
      - ./caddy-config:/config
      - ./caddy-data:/data
      - ./certificate.crt:/etc/caddy/certificate.crt
      - ./private.key:/etc/caddy/private.key
    environment:
      DOMAIN: "TODO"                # 您的域名，以 http 或 https 作为前缀
      EMAIL: "TODO"                 # 用于 ACME 注册的电子邮件地址
      LOG_FILE: "./data/access.loig"

4、创建Caddyfile配置文件，内容不用修改。

vim Caddyfile

{$DOMAIN} {
  log {
    level INFO
    output file {$LOG_FILE} {
      roll_size 10MB
      roll_keep 10
    }
  }

  # 使用 ACME HTTP-01 验证方式为已配置的域名获取证书
  tls {$EMAIL}

  # 此设置可能会在某些浏览器上出现兼容性问题（例如，在 Firefox 上下载附件）
  # 如果遇到问题，请尝试禁用此功能
  encode zstd gzip

  # 将所有代理到 Rocket
  reverse_proxy vaultwarden:80 {
       # 把真实的远程 IP 发送给 Rocket，让 Vaultwarden 把其放在日志中
       # 这样 fail2ban 就可以阻止正确的 IP 了
       header_up X-Real-IP {remote_host}
  }
}

5、更新和运行

#更新
docker compose pull 
#运行
docker compose up -d && docker compose logs -f
#停止
docker compose down

7、打开网页IP:80，即可登录使用。注意，云服务器安全组需要设置端口号80。

8、问题1，网页This site can’t be reached

docker logs caddy-CONTAINER-ID
#{"level":"error","ts":1739312132.2342033,"logger":"tls.obtain","msg":"will retry","error":"[47.108.144.177] Obtain: subject does not qualify for a public certificate: 47.108.144.177","attempt":12,"retrying_in":21600,"elapsed":21600.05246929,"max_duration":2592000}

原因是Caddy采用Let’s Encrypt获取公共 TLS 证书，而采用IP 地址不满足 Let’s Encrypt 的颁发条件，国内必须采用备案域名。

9、问题2，网页端登录，后台报错

在 docker 的日志中体现为 404:  
[2024-03-27 02:28:48.758][request][INFO] POST /identity/accounts/prelogin  
[2024-03-27 02:28:48.758][response][INFO] 404 Not Found

原因：参考link，Vaultwarden版本号原因，改为vaultwarden/server:latest改为vaultwarden/server:1.30.5
